## 1. 概念模型

```
Session（会话）1 ── * QARecord（问答记录)
   │                      │
   │  session_id (PK)     │  id (PK)
   │  title               │  session_id (FK → Session)
   │  query_count         │  query / answer / llm_used
   │  created_at          │  response_time_ms / answer_source
   └  updated_at          └  timestamp

Report（研报）独立实体
   │
   │  report_id (PK)
   │  title / filename / file_path
   │  company / industry / institution
   │  status / uploaded_at / analyzed_at
   └  analysis_result
```

> **核心关系**：
> - 一个 Session 可包含零到多条 QARecord。外键通过 `session_id` 字段逻辑引用。
> - Report 为独立实体，用于研报管理和分析。

## 2. 存储引擎

| 项 | 说明 |
|----|------|
| 引擎 | JSON 文件（`json` 标准库） |
| 编码 | UTF-8，缩进 2 空格 |
| 读写模式 | RMW（全量读入 → 修改 → 全量写回） |
| 并发安全 | 无锁，不支持并发写入 |
| 文件布局 | `{DATA_DIR}/sessions.json` + `qa_records.json` + `reports.json` |
| 文件存储 | `{DATA_DIR}/uploads/` 目录存储上传的研报文件 |

## 3. 实体定义 — Session

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `session_id` | `string` (UUID) | 是 | **PK** | 会话唯一标识 |
| `title` | `string` | 是 | ≤23 字符 | 默认"新会话"，首次问答自动重命名 |
| `created_at` | `string` (ISO-8601) | 是 | UTC+Z | 创建时间 |
| `updated_at` | `string` (ISO-8601) | 是 | UTC+Z | 最后更新时间 |
| `query_count` | `integer` | 是 | ≥ 0 | 累计问答次数 |

## 4. 实体定义 — QARecord

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `id` | `string` | 是 | **PK**，`rec_{ts}` | 基于时间戳生成 |
| `session_id` | `string` | 是 | **FK** → Session | 所属会话 |
| `query` | `string` | 是 | 1–500 字符 | 用户提问原文 |
| `answer` | `string` | 是 | — | LLM/Demo 返回的答案 |
| `llm_used` | `boolean` | 是 | RULE-005 | 是否使用真实 LLM |
| `model` | `string\|null` | 是 | — | 模型标识 |
| `response_time_ms` | `integer` | 是 | SLO < 5000 | 响应耗时（ms） |
| `answer_source` | `string\|null` | 否 | 枚举 | `copaw`\|`bailian`\|`demo` |

## 5. 实体定义 — Report

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `report_id` | `string` (UUID) | 是 | **PK** | 研报唯一标识 |
| `title` | `string` | 是 | ≤100 字符 | 研报标题，默认使用文件名 |
| `filename` | `string` | 是 | — | 原始文件名 |
| `file_path` | `string` | 是 | — | 文件存储路径 |
| `file_type` | `string` | 是 | 枚举 | `pdf`\|`doc`\|`docx` |
| `status` | `string` | 是 | 枚举 | `pending`\|`analyzing`\|`analyzed`\|`failed` |
| `company` | `string\|null` | 否 | — | 公司名称（分析后填充） |
| `industry` | `string\|null` | 否 | — | 行业标签（分析后填充） |
| `institution` | `string\|null` | 否 | — | 发布机构（分析后填充） |
| `uploaded_at` | `string` (ISO-8601) | 是 | UTC+Z | 上传时间 |
| `analyzed_at` | `string\|null` | 是 | UTC+Z | 分析完成时间 |
| `analysis_result` | `object\|null` | 否 | — | 分析结果，包含提取的实体和摘要 |

## 7. 关键业务逻辑

**示例 — 首次问答自动命名**：

```python
if session["query_count"] == 1:
    session["title"] = query[:20] + ("..." if len(query) > 20 else "")
```

> **规则**：仅首次问答（query_count 从 0 → 1）触发，后续不再自动更名。

| 逻辑 | 触发条件 | 行为 |
|------|----------|------|
| 首次问答自动命名 | `query_count` 从 0 → 1 | 标题 = query 前 20 字 + `...` |
| 级联删除 | 删除 Session | 同步删除所有关联 QARecord |
| 更新会话时间 | 新增问答记录 | 同步更新 Session.updated_at 为当前时间 |
| query_count 自增 | 新增问答记录成功 | Session.query_count += 1 |
| 记录 ID 生成 | 新增问答记录 | id = `rec_{timestamp}_{random}` 确保唯一 |
| 时间戳标准化 | 所有写操作 | 统一使用 ISO-8601 格式（UTC+Z）存储时间 |
| 导出文件生成 | 调用 export 接口 | 生成临时文件，返回下载链接（5分钟有效）|
| 研报文件存储 | 上传研报 | 文件保存至 `{DATA_DIR}/uploads/{report_id}.{ext}` |
| 研报分析状态流转 | 调用 analyze 接口 | `pending` → `analyzing` → `analyzed`/`failed` |
| 研报元数据提取 | 分析完成 | 提取 company/industry/institution 并更新 |
| 研报文件删除 | 删除研报 | 同步删除物理文件和数据库记录 |

## 8. 交互行为详细规格

### 8.1 选中会话交互行为

| 维度 | 说明 |
|------|------|
| **触发条件** | 用户点击 Sidebar 中的任意会话项（session item） |
| **API 调用** | `GET /api/sessions/{session_id}/records` |
| **请求参数** | `session_id`: 会话唯一标识（路径参数） |
| **成功行为** | ① 当前会话项高亮显示（添加 active 样式）<br>② 主内容区切换至 C 状态（对话历史列表）<br>③ 渲染该会话的所有问答记录（按时间正序）<br>④ 输入框 placeholder 更新为"继续提问..." |
| **失败行为** | 显示错误提示"加载会话记录失败"，保持当前状态 |
| **边界情况** | 若会话无记录，主内容区显示 B 状态（常见问题网格） |

### 8.2 B 状态 — 常见问题网格

| 元素 | 说明 |
|------|------|
| **标题** | "您可以这样问我" |
| **布局** | 2列网格布局，每个问题为一个可点击卡片 |
| **预设问题** | ① "分析新能源行业最新趋势"<br>② "对比宁德时代和比亚迪的财务指标"<br>③ "半导体行业本周研报观点汇总"<br>④ "医药板块投资建议有哪些" |
| **交互行为** | 点击任意问题卡片 → 自动填入输入框并聚焦 |
| **样式** | 卡片带悬停效果，浅蓝色边框，圆角 8px |

### 8.3 C 状态 — 对话历史卡片元素

| 元素 | 类型 | 说明 |
|------|------|------|
| **用户问题区** | 气泡卡片 | 右上角显示，浅蓝色背景，显示 query 内容 |
| **AI 回答区** | 文本块 | 左侧显示，白色背景，支持 Markdown 渲染 |
| **来源标签** | 徽章 | 根据 `answer_source` 显示不同颜色标签 |
| **时间戳** | 文本 | 回答生成时间，格式"HH:mm" |
| **响应时间** | 文本 | 显示"响应耗时 {response_time_ms}ms" |
| **操作按钮组** | 按钮组 | 复制、导出按钮 |

### 8.4 记录卡片中的来源标签逻辑

> 根据 `answer_source` 和 `llm_used` 组合显示不同标签。

| `answer_source` | `llm_used` | 显示文本 | 样式 |
|-----------------|------------|----------|------|
| `copaw` | `true` | CoPaw | 蓝色标签 |
| `bailian` | `true` | 百炼 | 绿色标签 |
| `demo` | `false` | 离线演示 | 灰色标签 |

### 8.5 记录卡片操作按钮

> 每条问答记录卡片右上角显示操作按钮组。

| 按钮 | 功能 | 触发行为 |
|------|------|----------|
| 复制 | 复制当前记录的 query + answer 到剪贴板 | 点击后显示"已复制"提示 |
| 导出 | 导出整个会话的所有记录 | 点击后弹出格式选择（JSON/TXT）→ 下载文件 |

### 8.6 研报管理页面交互行为

| 功能 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 加载研报列表 | 页面渲染 | `GET /reports` | 渲染研报卡片列表，显示状态标签 |
| 上传研报 | 点击"上传研报" | `POST /reports` | 弹出文件选择 → 上传 → 刷新列表 |
| 预览研报 | 点击预览按钮 | `GET /reports/<id>` | 弹出详情弹窗，展示研报元数据 |
| 删除研报 | 点击删除按钮 | `DELETE /reports/<id>` | 确认后删除，刷新列表 |
| 分析研报 | 点击分析按钮 | `POST /reports/<id>/analyze` | 状态变为 analyzing → analyzed |
| 筛选研报 | 点击筛选条件 | `GET /reports?status=xxx&industry=xxx` | 按状态/行业/时间筛选 |
| 搜索研报 | 输入搜索词 | 前端过滤 | 按标题/公司/行业实时过滤 |

**研报卡片元素**：

| 元素 | 说明 |
|------|------|
| 文件图标 | PDF/DOC/DOCX 类型标识 |
| 标题 | 研报标题，过长时截断 |
| 状态标签 | pending(待分析)/analyzing(分析中)/analyzed(已分析)/failed(失败) |
| 公司/行业标签 | 分析完成后显示 |
| 操作按钮 | 分析(待分析时)/预览/删除 |
