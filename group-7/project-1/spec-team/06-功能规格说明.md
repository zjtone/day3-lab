## 2. Header 区域

> 描述 Header 包含哪些元素。重点关注能力状态芯片（对齐 `05` US-008）。

### 2.1 能力状态芯片（Capability Chips）

芯片数据来源：挂载时调用 `GET /capabilities`，填充 `capabilities` state。

| 条件 | 芯片显示 | 样式 |
|------|----------|------|
| `caps.copaw_configured = true` | CoPaw 桥接 | 绿色边框 |
| `caps.bailian_configured = true` | 百炼 · {model} | 蓝色边框 |
| 两者均 false | 离线演示 | 灰色边框 |

## 3. Sidebar 会话管理

> 对齐 `05` US-001（新建）、US-002（查看/切换）、US-003（删除）、US-005（自动命名）。

| 功能 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 加载列表 | 页面渲染 | `GET /sessions` | 按 created_at 倒序渲染 + 默认选中第一个（US-002 AC-001） |
| 新建会话 | 点击"+ 新建" | `POST /sessions` | 插入列表头部 + 自动选中（US-001 AC-002） |
| 删除会话 | 点击 × | `DELETE /sessions/<id>` | 确认弹窗 → 确认后级联删除 → 列表刷新（US-003 AC-03/04） |
| 选中会话 | 点击列表中某条会话 | `GET /sessions/<id>/records` | 高亮选中行，Main 区加载该会话问答记录（US-002 AC-02） |
| 会话自动命名 | 首次问答成功后 | — | 当 `query_count` 从 0→1 时，标题更新为 query 前 20 字（US-005 AC-001） |

## 4. Main 内容区（三态渲染）

> 对齐 `05` US-001（问答提交）、US-002（历史记录查看）、US-006（降级标签）。描述三种条件下显示什么。

| 状态 | 条件 | 显示内容 |
|------|------|----------|
| **A 空状态** | `currentSession === null` | "请创建或选择一个会话开始" |
| **B 常见问题** | 有会话 && 无记录 | 显示 4~6 个预设常见问题卡片（如“某股的最新评级是什么？”），点击卡片自动填入输入框并发送提问 |
| **C 对话历史** | 有记录 | 正序渲染问答卡片，每张包含：用户提问气泡、AI 回答气泡、来源标签（CoPaw/百炼/离线演示）、响应耗时（ms）、原文引用折叠区（citations）、导出按钮（US-004） |

### 4.1 记录卡片中的来源标签逻辑

> 根据 `answer_source` 和 `llm_used` 组合显示不同标签。

| `answer_source` | `llm_used` | 显示文本 | 样式 |
|-----------------|------------|----------|------|
| `copaw` | `true` | CoPaw | 绿色/蓝色标签 |
| `bailian` | `true` | 百炼 | 绿色/蓝色标签 |

## 5. 输入区域

> 对齐 `05` US-002 AC（query 非空且长度 1~500）、US-004（导出触发）。

| 元素 | 类型 | 行为 |
|------|------|------|
| query-input | `<textarea>` | 3 行，placeholder "请输入您的问题..."，Ctrl+Enter 快捷发送 |
| send-btn | `<button>` | loading 时 disabled + 文案"发送中…"；完成后追加记录到 records |
| clear-btn | `<button>` | 清空输入框 |

### 5.1 导出功能（US-004）

| 触发 | 位置 | 行为 |
|------|------|------|
| 点击"导出"按钮 | C 对话历史卡片区右上角 | 弹出格式选择（JSON / TXT）→ 调用 `GET /sessions/<id>/export?format=json\|txt` → 浏览器下载临时链接（5分钟有效） |

## 6. 错误处理

| 后端 `error.code` | HTTP | 前端展示 |
|--------------------|------|----------|
| `EMPTY_QUERY` | 400 | 错误提示："请输入问题" |
| `INVALID_QUERY` | 400 | 错误提示："问题过长" |
| `SESSION_NOT_FOUND` | 400 | 错误提示：“会话不存在或已被删除”，自动刷新会话列表 |
| `INVALID_SESSION_ID` | 400 | 错误提示：“请先选择一个会话” |
| `INVALID_FILE_FORMAT` | 400 | 错误提示：“仅支持 PDF 和 HTML 格式的研报” |
| `INTERNAL_ERROR` | 500 | 错误提示：“服务异常，请稍后重试”，显示 traceId 便于排查 |

## 7. 状态管理

> React `useState` Hooks，列出你的组件需要管理哪些状态。

| State 变量 | 类型 | 初始值 | 用途 |
|------------|------|--------|------|
| `sessions` | `Session[]` | `[]` | 会话列表 |
| `currentSession` | `Session\|null` | `null` | 当前选中会话 |
| `records` | `QARecord[]` | `[]` | 当前会话的问答历史记录 |
| `query` | `string` | `""` | 输入框当前内容 |
| `loading` | `boolean` | `false` | 是否正在等待 AI 回答 |
| `error` | `string\|null` | `null` | 错误提示消息 |
| `capabilities` | `object\|null` | `null` | 系统能力状态（来自 GET /capabilities） |
| `showDeleteConfirm` | `boolean` | `false` | 是否显示删除确认弹窗 |

## 7. 研报管理页面

> 对齐 `05` US-007（研报上传、分析、预览、筛选）。

### 7.1 页面布局

```
┌─────────────────────────────────────────────┐
│  头部：标题"研报管理" + [上传研报按钮]          │
├─────────────────────────────────────────────┤
│  搜索筛选区：[搜索框] [筛选按钮]               │
├─────────────────────────────────────────────┤
│  研报列表区：                                 │
│  ┌─────────────────────────────────────┐   │
│  │ [图标] 研报标题          [状态标签]  │   │
│  │ 机构 · 日期              [分析][预览][删除]│
│  │ [公司标签] [行业标签]                  │   │
│  └─────────────────────────────────────┘   │
│  ... 更多研报卡片                           │
└─────────────────────────────────────────────┘
```

### 7.2 研报卡片状态

| 状态 | 显示 | 操作 |
|------|------|------|
| `pending` | 灰色标签"待分析" | 分析按钮可用 |
| `analyzing` | 橙色标签"分析中" | 分析按钮 loading |
| `analyzed` | 绿色标签"已分析" | 分析按钮隐藏 |
| `failed` | 红色标签"分析失败" | 可重新分析 |

### 7.3 筛选功能

| 筛选维度 | 选项 |
|----------|------|
| 状态 | 全部 / 已分析 / 分析中 / 待分析 / 失败 |
| 行业 | 动态从研报数据中提取 |
| 时间 | 全部 / 今天 / 近7天 / 近30天 / 近一年 |

### 7.4 预览弹窗

点击预览按钮后弹出详情窗口，展示：
- 研报标题、文件名、文件类型
- 发布机构、作者、报告日期
- 分析结果摘要（如果已分析）
- 文件路径
