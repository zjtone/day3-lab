## 一、测试分层

> 共 4 层，写测试前先搞清楚每层验什么。

| 层级 | 工具 | 覆盖对象 | 说明 |
|------|------|----------|------|
| **L1 Unit** | pytest | 单个函数/类的逻辑 | 不依赖网络，最快 |
| **L2 Integration** | pytest + Flask test_client | API 端点 + 存储 | 模拟 HTTP 请求 |
| **L3 Contract** | pytest + schema 断言 | 响应体结构与 `09` 一致 | 验字段名、类型 |
| **L4 E2E** | Playwright / Cypress | 完整用户旅程 | 浏览器级操作 |

## 二、用例拆分原则

> **核心理念：数"可独立失败的点"** — 每条 TC 只回答一个问题，失败时不看代码就知道哪里坏了。

| 来源 | 计数方法 | 示例 |
|------|----------|------|
| 正常路径 | 每个端点的 happy path = 1 条 | `POST /ask → 200` = TC-M01-001 |
| 错误码 | 每个 distinct error code = 1 条 | `EMPTY_QUERY` = TC-M01-002 |
| 分支路径 | 每个 if/else 降级分支 = 1 条 | 无 API Key → demo = TC-M01-004 |
| 类型约束 | Spec `09` 约定的字段类型 = 1 条 | `llm_used` 必须 bool = TC-M01-006 |

## 四、质量门禁

| Gate-ID | 条件 | 阈值 | 阻塞合并 |
|---------|------|------|----------|
| **G-LINT** | `black --check` + `flake8` 通过 | 无违规 | **是** |
| **G-UNIT** | Unit 测试全绿 | 100% 通过 | **是** |
| **G-INT** | Integration 测试全绿 | 100% 通过 | **是** |
| **G-AC** | P0 验收标准全覆盖 | 100% 覆盖 | **是** |
| **G-PERF** | `response_time_ms < 500` | Demo 模式 ≤ 500ms | 否（warning） |

### 执行命令

```bash
# 后端测试
cd backend
python -m pytest tests/ -q

# 后端测试 + 覆盖率
python -m pytest tests/ -q --cov=. --cov-report=term-missing

# 前端测试
cd frontend
npm run test
```

## 五、TDD 适用场景

| 场景 | TDD 价值 | 建议 |
|------|----------|------|
| 简单 CRUD（列表、删除） | 低 | 写完补测试 |
| 参数校验 / 错误码 | 中 | 建议 TDD |
| 多级降级 / 状态机 | **高** | **强烈建议 TDD** |
| 会话自动命名逻辑 | **高** | **建议 TDD** |
