## 1. 愿景与目标

构建投研问答助手，帮助用户在投研助手内完成研报查询、信息抽取、问答交互的全流程，实现研报归类整理、研报信息抽取、归类比对、自然语言交互的问答助手。

| 03 痛点 | 04 目标 |
|---------|----------|
| 信息分散、无统一入口 | 统一会话问答入口，支持新建会话并提问 |
| 研报较多，难以比对，关联分析比对困难 | 支持多研报对比查询与观点归类，降低人工比对成本 |
| 研报信息多、格式乱，需总结提取 | 自动解析研报并抽取关键指标，统一展示格式 |
| 缺乏溯源：问答结果无法追溯至原文出处 | 每条回答均展示来源、页码/段落位置及原文片段，确保可追溯性 |
| 历史记录难管理 | 会话历史自动沉淀，支持查看、切换、删除，随时回看 |

## 2. 目标用户

| 角色 | 使用目标 | 频率 |
|------|----------|------|
| 投研分析师 | 快速查询投研观点与市场解释，进行研报整理、指标抽取与归类汇总 | 每天多次 |
| 基金经理 | 通过问答快速获取研报观点摘要与对比结论，辅助投资决策 | 每天多次 |
| 数据分析实习生 | 借助 AI 问答完成研报整理、指标抽取与归类汇总任务 | 每天数次 |

## 3. 核心业务场景

**SC-01 新建会话并提问**：用户点击"新会话" → 输入问题 → 获得回答（含来源标识）→ 回答自动落库

**SC-02 查看与切换历史会话**：用户打开侧边栏 → 浏览会话列表 → 点击历史会话 → 加载该会话问答记录

**SC-03 删除会话**：用户在会话列表选中目标会话 → 点击删除 → 系统级联删除会话及其所有问答记录 → 列表刷新

**SC-04 研报信息抽取与对比**：用户输入研报内容或关键词 → AI 自动解析抽取关键指标 → 返回结构化结论并标注来源

**SC-05 问答记录导出**：用户选择导出格式（JSON/TXT）→ 系统生成临时下载链接 → 用户下载会话记录文件

| 编号 | 场景名 | 用户路径 |
|------|--------|----------|
| SC-01 | 新建会话并提问 | 用户点击"新会话" → 输入问题 → 获得流式回答（含来源标识）→ 回答自动落库 |
| SC-02 | 查看与切换历史会话 | 用户打开侧边栏 → 浏览会话列表 → 点击历史会话 → 加载该会话问答记录 |
| SC-03 | 删除会话 | 用户在会话列表选中目标会话 → 点击删除 → 系统级联删除会话及其所有问答记录 → 列表刷新 |
| SC-04 | 研报信息抽取与对比 | 用户输入研报内容或关键词 → AI 自动解析抽取关键指标 → 返回结构化结论并标注来源 |
| SC-05 | 问答记录导出 | 用户选择导出格式（JSON/TXT）→ 系统生成临时下载链接 → 用户下载会话记录文件 |
| SC-06 | 系统能力状态查看 | 用户查看Header能力状态芯片 → 了解当前LLM配置状态（CoPaw/百炼/离线演示） |

## 4. 产品规则

| 编号 | 规则 | 来源 |
|------|------|------|
| R-01 | 每条提问必须绑定 `session_id` | 03 §2 "会话式问答" |
| R-02 | `query` 长度 1～500 字符，为空或超长返回明确错误 | 03 §4 "P0 全部可测" |
| R-03 | 每条回答必须携带 `answer`、`llm_used`、`answer_source`、`response_time_ms` 字段 | 09 API §POST /ask 响应体 |
| R-04 | LLM 调用按三级降级顺序执行：CoPaw → 百炼（DashScope）→ Demo 离线兜底；`answer_source` 标识当前生效层级 | 03 §5 风险·外部LLM不可用 |
| R-05 | 会话创建后自动生成 `session_id`；删除会话时级联删除该会话下所有问答记录；`query_count` 随每次成功问答自动自增 | 03 §3 范围·会话管理 |
| R-06 | 首次问答后会话标题自动更新为query前20字，仅当query_count从0→1时触发 | 03 §3 范围·会话管理 |
| R-07 | 导出文件格式支持JSON和TXT，下载链接5分钟有效 | 03 §4 成功标准·问答导出 |
| R-08 | 能力状态探测返回copaw_configured和bailian_configured状态，用于Header芯片展示 | 09 API §GET /capabilities |

## 5. 能力与优先级

| 子能力 | 优先级 | 对应场景 |
|--------|--------|----------|
| 会话管理（列表 / 新建 / 删除） | **P0** | SC-01, SC-02, SC-03 |
| 问答提交与三级降级 | **P0** | SC-01 |
| 问答历史记录查看 | **P0** | SC-02 |
| 问答记录导出（JSON/TXT） | **P0** | SC-05 |
| 会话自动命名（首次问答） | **P0** | SC-01 |
| 研报关键指标抽取与对比归类 | **P1** | SC-04 |
| 系统健康状态检测（能力探测） | **P1** | SC-06 |

## 6. 验收与成功标准（PRD 层）

| 标准 | 度量 |
|------|------|
| 可演示 | 种子数据 ≥ 10 条下，主路径 + 历史 + 能力状态 **3 分钟内**演示完毕 |
| 离线可用 | 未配置密钥时系统仍返回离线演示回复（`answer_source=demo`） |
| 降级兜底可用 | 未配置任何 LLM 密钥时，问答仍返回有效回复，`answer_source=demo`，响应时间 ≤ 500ms |
| 会话管理完整 | 支持创建、查看、删除会话，删除时级联删除问答记录，数据隔离正确 |
| 问答流式响应 | 支持流式响应展示，用户可实时看到AI回答生成过程 |
| 导出功能可用 | 支持将会话记录导出为JSON/TXT格式，下载链接5分钟有效 |
| 自动命名正确 | 首次问答后会话标题自动更新为query前20字 |
