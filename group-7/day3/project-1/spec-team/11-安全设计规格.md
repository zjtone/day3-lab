## 1. 安全目标与合规要求

> 明确本模块需遵循的安全目标和外部合规标准。投研场景尤其关注数据隐私与内部信息防泄露。

| 安全目标 | 说明 | 对应合规 |
|----------|------|----------|
| 数据不出域 | 所有用户数据、研报内容、问答记录禁止传输至外部服务 | 内部信息安全规范 |
| 最小权限原则 | 每个组件只拥有完成任务所需的最低权限 | 安全设计最佳实践 |
| 可追溯性 | 所有用户操作与数据访问需记录审计日志 | 内部审计规范 |
| 会话数据隔离 | 不同用户的会话数据严格隔离，防止越权访问 | ← 05 US-002 AC-002-04 |
| 降级可用性 | 外部服务不可用时系统仍能安全降级运行 | ← 03 §5 风险·外部LLM不可用 |

> 定义用户身份验证方式和权限控制模型。写出认证流程和角色划分。

### 2.1 认证机制

| 项目 | 规格 |
|------|------|
| 认证方式 | 无（Workshop 教学版开放访问） |
| Token 有效期 | 不适用 |
| 密码策略 | 不适用 |

| 访客 | ✗ | ✗ | ✗ | ✗ | ✗ |

## 3. 数据安全

> 定义数据分类分级、加密策略、存储安全规范。**投研场景核心关注点**。

### 3.1 数据分类与分级

| 数据类型 | 安全等级 | 示例 | 保护措施 |
|----------|----------|------|----------|
| 研报原文 | **机密** | PDF/HTML 研报文件 | 加密存储，访问审计 |
| 问答记录 | **内部** | 用户提问与AI回答 | 存储加密，定期清理 |
| 用户信息 | **内部** | 用户名、角色、操作日志 | 脱敏展示，最小采集 |
| 系统配置 | 一般 | 模型参数、界面配置 | 版本控制 |

### 3.2 加密策略

| 场景 | 加密方式 | 说明 |
|------|----------|------|
| 传输加密 | HTTP（教学环境） | Workshop 内网环境，生产环境需 HTTPS |
| 存储加密 | 无 | 本地 JSON 文件明文存储，生产环境需加密 |
| 密钥管理 | 环境变量 `.env` | 禁止硬编码，生产环境需 KMS |
| 当前实现 | 教学版暂不使用加密，数据以明文JSON存储于 `backend/data/` 目录 | |
| 生产建议 | 1. 启用HTTPS/TLS 1.3传输加密<br>2. 敏感数据字段级加密（AES-256）<br>3. 使用KMS管理API密钥<br>4. 数据库连接SSL加密 |

## 4. 接口安全

> 与 `09` API 规格配合，定义 API 层面的安全防护措施。

| 防护措施 | 实现方式 | 说明 |
|----------|----------|------|
| 输入校验 | 服务端白名单校验 | 所有用户输入进行类型、长度、格式校验 |
| SQL 注入防护 | 不适用 | 使用JSON文件存储，无SQL数据库 |
| XSS 防护 | 输出转义 + CSP | 问答内容渲染时防止脚本注入 |
| CSRF 防护 | 无 | 教学环境暂不启用，生产环境需 Token 校验 |
| 速率限制 | 无 | 教学场景不限速，生产环境需限制 |
| 文件上传校验 | 类型白名单 + 大小限制 | 仅允许 PDF/HTML，限制单文件大小 |
| 当前实现说明 | 教学版API开放访问，未启用CSRF防护；生产环境建议：<br>1. 启用CSRF Token验证<br>2. 配置SameSite Cookie属性<br>3. 添加请求来源校验（Referer/Origin） | |

## 5. LLM / AI 安全

> 针对大模型集成的专项安全措施。投研场景数据敏感，需严格管控数据流向。

| 风险点 | 防护措施 | 验收标准 |
|--------|----------|----------|
| 数据外泄至第三方模型 | 本地部署模型 / 内网百炼服务，禁止调用外网 API | 网络出口审计零外传记录 |
| Prompt 注入攻击 | 系统 Prompt 隔离，用户输入预处理过滤 | 敏感指令过滤率 100% |
| 模型幻觉 / 错误信息 | 回答标注来源引用，标识置信度 | 所有回答携带 `answer_source` 标识 |
| 敏感信息泄露（模型回复） | 输出过滤层，正则匹配敏感信息并脱敏 | 身份证号、手机号等敏感信息脱敏率 100% |
| 降级安全 | 三级降级策略确保服务可用性 | ← 04 §4 R-04 / 05 US-006 AC-006-01 |

> **验收标准补全**：
> - **AC-LLM-01**：`answer_source` 字段必须正确标识当前使用的LLM层级（copaw/bailian/demo）
> - **AC-LLM-02**：未配置任何外部LLM密钥时，系统自动降级至Demo模式，响应时间 ≤ 500ms
> - **AC-LLM-03**：Demo模式返回的回复内容不得包含敏感信息或真实业务数据
> - **AC-LLM-04**：LLM调用失败时系统不暴露内部错误详情，仅返回友好提示

## 6. 审计与日志

> 定义安全审计日志的记录范围、保留策略。

| 事件类型 | 记录内容 | 保留期限 | 告警 |
|----------|----------|----------|------|
| 登录 / 登出 | 用户ID、时间、IP、结果 | 180天 | 连续失败 ≥5 次告警 |
| 研报访问 | 用户ID、文档ID、操作类型 | 365天 | 异常批量下载告警 |
| 数据导出 | 导出内容摘要、目标 | 365天 | 每次导出需审批 |
| 系统配置变更 | 变更人、变更前后值 | 永久 | 关键配置变更即时通知 |
| 会话操作 | 会话创建/删除/切换记录 | 90天 | 异常频繁操作告警 |
| 问答记录 | 查询时间、会话ID、回答来源 | 90天 | 无 |

## 7. 安全威胁清单（STRIDE 简表）

> 基于 STRIDE 模型，列出主要安全威胁与应对方案。

| 类型 | 威胁 | 风险等级 | 应对措施 | 负责人 |
|------|------|----------|----------|--------|
| S | 伪造用户身份访问会话数据 | **高** | 会话数据绑定用户标识，访问时校验归属 | 开发负责人 |
| T | 问答记录在传输中被篡改 | 中 | 生产环境启用HTTPS/TLS | 后端负责人 |
| R | 会话列表信息被未授权读取 | **高** | 用户只能访问自己的会话列表（← 05 US-002 AC-002-04） | 开发负责人 |
| I | 内部数据通过AI模型泄露 | **高** | 本地部署 + 输出过滤，禁止调用外网API | AI负责人 |
| D | 大量请求导致服务不可用 | 中 | 速率限制 + 降级策略（← 04 §4 R-04） | 运维负责人 |
| E | 未授权访问会话数据 | **高** | 会话数据隔离，仅允许访问本人会话（← 05 US-002 AC-002-04） | 开发负责人 |

## 8. 部署安全

> 定义部署架构中的安全要求。

- [x] 生产/开发环境**严格隔离**，数据不互通
- [x] 敏感配置（API Key、密码）通过环境变量注入，不入代码仓库
- [ ] 数据库访问仅限应用服务器 IP 白名单（教学版使用JSON文件存储）
- [ ] 依赖包定期安全扫描（`pip audit` / `npm audit`）
- [ ] 容器镜像安全扫描，无高危漏洞
- [ ] 运行时安全监控，异常行为告警

> **补充部署安全要求**：
> - [x] `.env` 文件已加入 `.gitignore`，禁止提交到代码仓库
> - [x] 后端服务绑定 `127.0.0.1` 或内网IP，不直接暴露公网
> - [ ] 生产环境启用WAF（Web应用防火墙）防护
> - [ ] 定期备份 `backend/data/` 目录数据，保留最近7天备份
> - [ ] 生产环境配置日志集中收集与分析（ELK/Loki）
> - [ ] API接口启用请求日志记录，便于安全审计
> - [ ] 敏感操作（删除会话、导出数据）需二次确认
