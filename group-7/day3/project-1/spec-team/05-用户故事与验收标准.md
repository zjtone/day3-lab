> **本文定位**：按用户能理解的子场景拆成多条 **User Story**，并为每条写清 **REQ-ID**、**验收标准（AC）**。

## 字段说明 — 每条 UserStory 必须包含以下字段

| 字段 | 填写说明 | 数据来源 ← | 流向下游 → |
|------|----------|------------|-----------|
| **需求编号** | 唯一编号，如 REQ-M1QA-001 | ← `02` 采集编号 | → `14` 追踪矩阵首列 |
| **故事** | 作为【角色】，我希望【行为】，以便【价值】 | ← `04` PRD 场景拆解 | → `06` FSD 功能点 |
| **优先级** | P0（必须）/ P1（应该）/ P2（可选） | ← `03` §4 成功标准 | → `12` 里程碑排期 |
| **来源** | 对应 03/04 哪一条需求 | ← `03` 范围 · `04` 场景 | |
| **前置条件** | 触发该故事的前提状态 | ← `04` 用户流程 | → `13` 测试前置 |
| **功能对齐** | 对应功能规格 + 非功能约束 | | → `06` FSD · `07` NFR |
| **设计对齐** | 对应架构、接口、数据、安全 | | → `08` 架构 · `09` API · `10` Data · `11` 安全 |
| **验证对齐** | 对应测试用例 + 追踪矩阵行 | | → `13` TC · `14` 追踪矩阵行 |
| **验收标准** | AC，每条可独立验证 | ← 故事 + `07` NFR | → `13` 测试用例依据 |

## US-001（P0）新建会话并提问

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M1QA-001 |
| **故事** | 作为投研分析师，我希望点击"新会话"并输入问题，以便开启独立的AI对话任务并获得流式回答 |
| **优先级** | P0 |
| **来源** | ← 03 §3 提案、会话管理 / 04 §3 SC-01 |
| **前置条件** | 用户已登录系统 |
| **功能对齐** | → 06 §3 Sidebar · 06 §5 输入区 · 07 §1.2 |
| **设计对齐** | → 08 §2 · 09 POST /sessions, POST /ask API · 10 Session/QARecord表 · 11 §2 |
| **验证对齐** | → 13 TC-001 · 14 第1行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-001-01 | 支持创建新会话，新创建的会话独立，不携带其他会话的历史消息 |
| AC-001-02 | 创建会话后自动生成唯一的 `session_id` |
| AC-001-03 | 用户可在输入区输入问题（1-500字符），点击发送后向AI提交问题 |
| AC-001-04 | AI以流式方式响应回答，用户可实时看到回答生成过程 |
| AC-001-05 | 每条回答必须携带 `answer`、`llm_used`、`answer_source`、`response_time_ms` 字段 |
| AC-001-06 | 问答记录自动落库保存，包含query、answer、timestamp等信息 |

## US-002（P0）查看与切换历史会话

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M1QA-002 |
| **故事** | 作为投研分析师，我希望查看侧边栏中的历史会话列表并切换会话，以便回顾不同任务的问答记录 |
| **优先级** | P0 |
| **来源** | ← 03 §3 提案、会话管理 / 04 §3 SC-02 |
| **前置条件** | 已存在一个或多个会话 |
| **功能对齐** | → 06 §3 Sidebar · 06 §4 Main内容区 · 07 §5.1 |
| **设计对齐** | → 08 §2 · 09 GET /sessions, GET /sessions/{id}/records API · 10 Session/QARecord表 · 11 §2 |
| **验证对齐** | → 13 TC-002 · 14 第2行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-002-01 | 侧边栏展示当前用户的所有会话列表，按时间倒序排列 |
| AC-002-02 | 点击历史会话后，主内容区加载该会话的所有问答记录 |
| AC-002-03 | 不同会话的数据互相隔离，切换会话时仅显示当前会话的问答记录 |
| AC-002-04 | 用户只能查看本人的会话，无法查看其他用户的会话 |

| **故事** | 作为投研分析师，我希望首次问答后会话自动命名，以便快速识别会话主题 |
| **优先级** | P0 |
| **来源** | ← 03 §3 范围·会话管理 / 04 §4 R-06 |
