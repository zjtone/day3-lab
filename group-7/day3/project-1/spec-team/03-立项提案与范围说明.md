## 1. 背景与问题

> 描述用户遇到的问题，至少写 2~3 条痛点。参考：信息获取、历史记录、可追溯性、系统可用性。

| 痛点 | 来源 |
|------|------|
| 信息分散：研报、新闻、数据分布在多个系统，无统一问答入口 | 用户访谈 |
| 研报较多，难以比对，关联分析比对困难 | 用户访谈 |
| 研报信息多、格式乱，需总结提取 | 用户访谈 |
| 缺乏溯源：问答结果无法追溯至原文出处，难以验证准确性 | 用户访谈 |
| 历史记录难管理：过往问答无法有效保存和复用 | 用户访谈 |

---

## 2. 提案概要

> 用 3~5 句话概括要做什么。参考格式："在 XXX 应用内交付 YYY 能力"。要素：会话式问答 · 能力降级 · 可追溯

在投研助手内交付研报查询能力、信息抽取能力、问答能力，实现研报归类整理、研报信息抽取、归类比对、自然语言交互的问答助手。支持用户上传 PDF/Word 研报文件或粘贴研报文本，系统自动解析抽取标题、发布机构、日期、行业、公司、研究员等关键信息，并生成摘要。用户可基于上传的研报进行智能问答，系统提供流式响应并关联研报原文，每条回答均展示来源、页码/段落位置及原文片段，确保可追溯性。同时支持问答内容的收藏和纪要导出，方便投资经理和分析师后续使用。

## 3. 范围边界

> 明确"做什么"和"不做什么"，避免范围蔓延。各写 3~4 条。

| 做（范围内） | 不做（范围外） |
|-------------|---------------|
| 研报上传与入库：支持 PDF/Word 文件上传，自动解析入库 | 多租户/行级数据权限 |
| 研报自动解析：标题、发布机构、日期、行业、公司、研究员、自动生成摘要 | 新闻监控、公告查询、告警 |
| 标签体系：行业标签（新能源/半导体/消费/医药等）、类型标签（深度报告/公司点评/行业周报/业绩快报）、重要度标签 | 相关股票实时行情数据查询 |
| 研报检索：按公司名、行业、时间、机构筛选 | 研报实时推送、订阅功能 |
| 会话管理：创建、列表、删除会话 | 用户权限管理、组织架构集成 |
| AI问答流式响应、研报原文关联、回答溯源 | 多模态问答（图片、图表解析） |
| 关键指标抽取对比归类 | 复杂财务模型计算、估值分析 |
| 收藏问答、导出纪要功能 | 社交分享、协作编辑功能 |
| 登录和角色选择（投资经理/分析师） | SSO单点登录、LDAP集成 |

## 4. 成功标准（可度量）

> 写出 3~4 条可量化的验收标准。句式：用户在 X 分钟内完成 XXX 全链路 / 未配置外部模型时返回 XXX

| 标准 | 度量方式 |
|------|----------|
| 主路径可走通 | 用户在 3 分钟内完成"新建会话 → 提问 → 查看历史"全链路 |
| 研报信息抽取 | 上传研报文件或粘贴研报文本后，自动解析抽取，关键字段识别准确率 ≥ 80% |
| 研报对比查询 | 支持研报查询、对比、观点归类，响应时间 ≤ 2 秒 |
| 回答可追溯 | 每条 AI 回答均展示来源研报标题、发布机构、日期、页码/段落及原文片段 |
| 降级可用 | 未配置外部模型时返回 Demo 模式兜底响应，系统不崩溃 |
| 问答导出 | 支持将问答内容导出为纪要文件（Markdown/PDF 格式） |

## 5. 风险（简表）

> 列出 2~3 个主要风险。常见方向：外部依赖不可用、数据存储并发、模块间契约延期。

| 风险 | 影响 | 应对 |
|------|------|------|
| 外部 LLM 不可用 | 问答功能降级 | 三级降级：CoPaw → 百炼 → Demo 模式兜底 |
| AI未抽取出有效数据 | 部分研报无法解析内容 | 提供人工填写表单，由真人填写内容 |
| 研报解析格式复杂 | PDF/Word 格式多样，解析失败率高 | 优先支持常见格式，异常文件人工处理 |
| 数据存储并发问题 | JSON 文件存储在并发场景下可能冲突 | 采用文件锁机制或会话级隔离 |
| 开发时间受限 | 约 3 小时开发时间，功能需简化 | 采用 AI 生成模拟数据，聚焦核心功能 |
